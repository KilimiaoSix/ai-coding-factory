# 第 23 章：安全设计总结——纵深防御

## 问题定义

Claude Code 的安全性不是由某个单点机制保证，而是由权限、沙箱、输入验证、路径规则、UI 提示、会话审计等多层机制共同形成的纵深防御。

## 架构分析

从链路上看，安全至少经过这些层：模型看到的工具定义本身已被裁剪；Tool 输入先做 Schema 校验；运行前走权限规则；Shell 和文件路径再做专门分析；需要时走沙箱；执行结果进入 transcript 和 UI，可被追踪和审计。任何一层单独失败，都不应直接把高风险操作暴露给模型。

## 设计启示

- 安全设计最忌讳把希望寄托在某一个“万能检查”上。
- 对 Agent 来说，越靠近执行点的工具越需要多层防护。
- 审计与可见性本身就是安全边界的一部分。
