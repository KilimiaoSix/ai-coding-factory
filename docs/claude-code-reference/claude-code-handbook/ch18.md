# 第 18 章：子 Agent 调度——AgentTool 与任务委派

## 问题定义

当单个上下文不足以完成任务，Claude Code 会把部分工作委派给子 Agent。`AgentTool` 不是普通工具，它本质上是在当前会话内部再启动一个受控的 Agent 生命周期。

## 架构分析

`AgentTool` 负责：定义子 Agent 输入 Schema、选择内建或自定义 agent 类型、为子 Agent 组装工具池、决定前台/后台执行、在需要时创建 worktree 或 remote 隔离、把结果回收为父会话可消费的消息。围绕它还形成了颜色管理、任务输出、恢复与继续发送消息等配套机制。

## 设计启示

- 子 Agent 应当像其他 Tool 一样被声明、被约束，而不是做成系统外的特殊通道。
- 委派的关键不是“把任务丢出去”，而是让子任务拥有清晰的权限、上下文和回收路径。
- 多 Agent 协作越强，越需要显式生命周期管理。
