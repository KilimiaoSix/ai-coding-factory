# 第 41 章：OAuth 与认证架构

## 问题定义

Claude Code 的认证并不只是“有没有 API key”，它还涉及 Claude.ai OAuth、PKCE、本地回调监听、手动授权码输入、令牌刷新和用户资料同步。

## 架构分析

`OAuthService` 封装了完整的 OAuth 2.0 Authorization Code + PKCE 流程：启动本地监听器、生成 challenge/state、打开浏览器或交给上层展示 URL、拿到授权码后换取 token，再请求 profile 信息补全订阅与限额元数据。`client.ts` 则提供 token 刷新、scope 判断、资料拉取和本地配置同步能力。

- 认证状态会影响 GrowthBook、usage API、Bridge、Teleport、MCP 鉴权等多个子系统，因此不是一块独立孤岛。

## 设计启示

- 对桌面或终端 Agent，PKCE + localhost callback 是兼顾安全和体验的现实方案。
- 认证最好同时带出用户画像和权限上下文，而不只是 access token。
- 认证系统一旦进入多产品面向（console、Claude.ai、remote control），统一的 client 抽象就很重要。
