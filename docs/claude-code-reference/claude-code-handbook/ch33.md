# 第 33 章：输入系统——用户如何与 Agent 交互

## 问题定义

Claude Code 的输入框不仅接收纯文本，还要处理 slash command、历史搜索、Vim 模式、图片粘贴、快捷键、背景任务导航、权限模式切换和多 Agent 协作提示。

## 架构分析

输入系统由 `PromptInput`、`REPL`、keybinding 和若干 hooks 共同构成。`PromptInput` 管输入态本身，`REPL` 负责把它嵌入更大界面，keybinding 系统则把终端键盘事件翻译成“提交、滚动、切模式、打开对话框、进入搜索”等高层动作。

## 快照修正与补充

- `other-ans/ch33.md` 把输入系统理解为“用户如何随时介入 Agent 循环”，这是准确的；当前 `PromptInput` 文件体量本身就说明了输入系统在产品中的核心地位。
- 输入系统与权限模式、背景任务、技能推荐、多 Agent 视图等多个子系统耦合，因此它不能被当作普通文本框看待。
- 某些特性如语音、buddy、ultraplan 等会按 feature gate 或环境条件出现或隐藏。

## 设计启示

- 在 Agent 产品里，输入框是控制台，不只是文本编辑器。
- 键盘交互和模式切换设计得好，用户才会觉得 Agent 始终可控。
- 复杂输入系统最好围绕状态与事件编排，而不是堆叠局部 hack。
