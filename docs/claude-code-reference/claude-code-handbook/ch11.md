# 第 11 章：上下文组装——给 Agent 看什么

## 问题定义

上下文管理不只是“放不放得下”，还包括“应该给模型看什么”。系统提示、当前目录、Git 状态、MCP 服务器、记忆文件和最近对话都会影响模型行为。

## 架构分析

当前快照中的上下文组装呈现明显的模块化风格：静态提示词定义基础角色，动态提示词注入当前环境和会话状态，QueryEngine 再把这些片段与消息历史拼成一次调用。这样做的好处是每类上下文都能单独演进，而不会把一个巨型 prompt 绑死在某处。

## 设计启示

- 让提示词按来源分层，比维护一个超大 system prompt 更可控。
- 环境信息必须和权限边界一起提供，否则模型会看到无法执行的能力。
- 上下文组装是 QueryEngine 的核心竞争力之一，因为它决定模型“看见的世界”。
