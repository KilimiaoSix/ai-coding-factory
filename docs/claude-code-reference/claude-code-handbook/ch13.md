# 第 13 章：跨会话记忆（SessionMemory）

## 问题定义

上下文压缩解决的是“当前窗口不够”，SessionMemory 解决的是“会话结束之后还需要记住什么”。当前快照把记忆做成 Markdown 文件驱动的后台提炼机制，而不是藏在数据库里的黑盒状态。

## 架构分析

`src/services/SessionMemory/sessionMemory.ts` 会在满足阈值时触发记忆提取，通过 forked subagent 在后台整理当前会话的结构化摘要。系统区分了工作记忆、项目记忆和用户/长期记忆：SessionMemory 负责当前会话提炼，`memdir` 和 CLAUDE.md 则负责更长期的内容。

## 关键源码锚点

## 快照修正与补充

- `other-ans/ch13.md` 对模板字段和分层记忆讲得很细，本手册保留这条主线，但只把源码中能直接确认的部分视作事实。
- `../09-config-and-state.md` 提到“持久化内存”，而 SessionMemory 则给出了它在当前快照中的一个具体落点。
- 记忆提取依赖后台子任务和特性开关，外部快照里这套能力的可见性高于默认启用程度。

## 设计启示

- 记忆最好的形式往往是文件，因为它透明、可审计、可复用。
- 记忆更新应当在后台进行，避免打断主对话。
- 记忆系统必须有容量与触发阈值，否则会反过来污染上下文。
