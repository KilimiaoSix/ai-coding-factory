# 第 37 章：技能系统（Skills）

## 问题定义

技能是 Claude Code 中极具产品特色的一层：它不是普通命令，也不是普通插件，而是可被模型或用户调用的、带 frontmatter 元数据和提示内容的能力片段。

## 架构分析

技能来源包括 bundled skills、用户目录、项目目录、managed 目录、legacy commands 目录以及 MCP skill builders。`loadSkillsDir.ts` 负责发现、去重、解析 frontmatter、按路径条件激活、过滤 plugin-only 策略，并把技能转换为命令接口供系统统一消费。

## 关键源码锚点

- `src/skills/loadSkillsDir.ts`
- `src/skills/bundledSkills.ts`
- `src/skills/bundled/`
- `src/skills/mcpSkillBuilders.ts`
- `src/utils/markdownConfigLoader.ts`
- `src/commands/skills/`

## 快照修正与补充

- `other-ans/ch37.md` 把技能定义为“可复用提示词机制”，这在当前快照是准确的，但源码进一步说明技能同时还是一个受限命令对象。
- `../05-command-system.md` 只把技能当成命令来源之一，本章强调技能拥有自己的目录搜索、frontmatter 语义和动态激活逻辑。
- 技能不是无限制扩展点，依然受来源、frontmatter 校验、plugin-only 策略和用户显式启用影响。

## 设计启示

- 技能系统把“提示工程”从隐藏内部实现变成了可管理、可分发的资产。
- 用 frontmatter 把工具约束、触发条件和模型偏好显式化，是很强的工程化做法。
- 技能越多，去重、来源优先级和条件激活就越重要。
