# 第 22 章：沙箱——Agent 的活动围栏

## 问题定义

权限回答“能不能做”，沙箱回答“即使做了，影响范围能被限制到哪里”。在 Claude Code 里，沙箱不是单独的理论概念，而是与 Tool、权限规则、设置系统和平台差异相连的一层运行时适配器。

## 架构分析

`src/utils/sandbox/sandbox-adapter.ts` 负责把 Claude Code 自己的设置、路径规则和权限表达转换成外部 sandbox runtime 能理解的配置，包括文件读写限制、网络限制、例外规则和回调。UI 层还有 `src/components/sandbox/` 和 `/sandbox-toggle` 命令用于展示和配置沙箱状态。

## 关键源码锚点

- `src/utils/sandbox/sandbox-adapter.ts`
- `src/utils/sandbox/sandbox-ui-utils.ts`

## 快照修正与补充

- 当前外部快照缺失真实的 `@anthropic-ai/sandbox-runtime` 实现，`scripts/build.ts` 会为其提供桩。因此本章只能可靠描述适配层、设置转换和 UI，而不能把底层沙箱实现写成完整可验证事实。
- `docs/07-permission-system.md` 里把沙箱视为权限之外的一层，这个关系在当前快照中成立。
- `other-ans/ch22.md` 的价值主要在于说明“围栏式隔离”的架构意义；具体底层实现要以当前快照可见代码为边界。

## 设计启示

- 没有沙箱的 Agent 权限系统，只能防君子，难以兜住执行范围。
- 适配层比底层 runtime 更值得阅读，因为它解释了产品语义如何映射到系统约束。
- 对外部快照做文档时，必须明确区分“适配逻辑存在”和“底层能力可运行”。
