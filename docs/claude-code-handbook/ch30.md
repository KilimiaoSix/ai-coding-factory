# 第 30 章：对话历史的持久化与恢复

## 问题定义

如果 Agent 会话不能可靠地落盘和恢复，那么长任务、跨机器继续、崩溃恢复和历史审计都会变得脆弱。Claude Code 因此把 transcript 持久化做成了一套完整的 JSONL 体系。

## 架构分析

`src/utils/sessionStorage.ts` 负责写入、读取、裁剪和恢复 transcript；它区分真正参与会话链的消息与只用于 UI 的进度消息，并维护会话目录、版本、工作区、compact boundary、文件历史快照等附加信息。恢复时，系统既要拿回对话内容，也要修补旧格式和异常中断留下的边缘情况。

## 快照修正与补充

- `docs/09-config-and-state.md` 提到会话恢复，但没有展示 transcript 与 parentUuid 链、progress 过滤这些细节。
- 历史持久化在当前快照中也是 Analytics、Teleport、Resume、Compact 等多个系统的基础。

## 设计启示

- Transcript 是 Agent 系统最重要的事实日志之一。
- 恢复逻辑一定要兼容旧格式和中断场景，否则长期运行会话无法可靠维护。
- 对话历史不只是为了“继续聊天”，也是调试、审计和异步任务回收的基础。
