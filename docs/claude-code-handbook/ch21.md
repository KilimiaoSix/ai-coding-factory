# 第 21 章：权限的运行时执行

## 问题定义

权限模式和规则只是一套静态声明，真正困难的是运行时执行：工具请求来了之后，系统怎样评估、提示、缓存、更新状态，并与 UI 和远程审批协同。

## 架构分析

运行时执行大致分三步：先做规则引擎静态评估，再根据当前 Agent 类型和模式走运行时编排，最后在需要时进入用户交互或分类器审批。整个流程要处理拒绝追踪、建议规则、动态更新和多路竞速，避免既过度打扰用户，又让高风险操作漏过。

## 快照修正与补充

- `other-ans/ch21.md` 强调运行时“多路竞速交互”和 AI 分类器，这些概念在当前快照中有对应实现文件，但具体是否启用会受配置与远程环境影响。
- `docs/07-permission-system.md` 更像静态总览，本章强调的是“规则落地到一次工具执行”的细节。
- 分类器类路径属于“自动化批准的增强层”，不应被理解为权限系统的唯一入口。

## 设计启示

- 权限执行应当是一条明确的管线，而不是散落在各个工具里的 `if` 语句。
- 运行时审批越复杂，越需要统一的上下文和结果类型。
- 用户体验与安全性在权限执行阶段最容易产生张力，因此要把交互节奏也纳入设计。
