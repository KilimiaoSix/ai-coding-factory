# 第 34 章：虚拟滚动与渲染性能

## 问题定义

长会话终端 UI 最大的性能敌人不是单条消息，而是“所有消息同时存在且持续重绘”。Claude Code 因此在 REPL 与 Ink 层实现了虚拟滚动、ScrollBox、冻结离屏内容和多种滚动优化。

## 架构分析

`REPL.tsx` 中的 transcript 视图已经显式区分 virtual scroll 与普通 scrollback。`ScrollBox` 提供命令式滚动句柄，渲染器则利用 viewport culling、硬件滚动提示、pending scroll drain 和离屏冻结减少重排与 ANSI 输出量。对代码高亮和 Markdown，模块级缓存也被用于降低长会话 remount 成本。

## 快照修正与补充

- 当前优化目标不仅是省 CPU，还要避免大段 ANSI 输出导致的终端闪烁和延迟。

## 设计启示

- 终端 UI 一样需要窗口化和性能预算。
- 只优化 React 组件还不够，终端输出层本身也是性能瓶颈。
- 长会话产品如果不做虚拟滚动，最终会被自己的历史记录拖垮。
