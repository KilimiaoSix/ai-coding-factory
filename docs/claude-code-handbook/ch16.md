# 第 16 章：Shell 执行——Agent 最危险的工具

## 问题定义

Shell 是 Claude Code 最强也最危险的工具。一旦模型能运行命令，它就不仅能读写代码，还能影响进程、网络、Git 状态和外部系统，因此 BashTool 与 PowerShellTool 都被设计成多层安全链，而不是简单的 `spawn()` 包装。

## 架构分析

`BashTool` 和 `PowerShellTool` 都把能力拆成多份辅助模块：命令语义判断、只读验证、路径验证、破坏性警告、沙箱适配和权限规则匹配。真正的执行只是最后一步，前面的元数据和校验才是它们之所以能出现在 Agent 系统里的原因。

## 快照修正与补充

- `other-ans/ch16.md` 把 Shell 描述为“最危险的工具”，这一判断在当前快照中非常准确，因为与之配套的权限、路径和沙箱代码远多于其他 Tool。
- `docs/03-tool-system.md` 没有充分展示 Shell 的安全细节，本章用目录粒度把这些辅助模块补齐。
- PowerShell 支持是平台扩展的一部分，它在 `src/tools.ts` 中按条件注册，而不是默认与 Bash 完全对等。

## 设计启示

- Agent 的 Shell 能力必须默认假设为高风险。
- 对 Shell 来说，输入 Schema 只解决“格式正确”，安全仍要靠专门的语义分析与规则匹配。
- 终端 Agent 的威力往往体现在 Shell，但真正的工程质量也最容易在 Shell 这里暴露。
