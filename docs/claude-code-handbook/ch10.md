# 第 10 章：上下文窗口——Agent 最稀缺的资源

## 问题定义

终端 Agent 需要把系统提示、工具定义、对话历史、工具结果、记忆文件都塞进同一个上下文窗口。相比 CPU 或内存，token 预算才是 QueryEngine 最稀缺的资源。

## 架构分析

Claude Code 不是只有一个“超限就压缩”的粗糙策略，而是多层回收链：先预算工具结果，再做 snip、microcompact、context collapse、autocompact，最后才进入更重的恢复路径。这个顺序本质上是在用更便宜的手段优先释放 token。

## 关键源码锚点

- `src/query.ts`
- `src/services/compact/autoCompact.ts`
- `src/services/compact/microCompact.ts`
- `src/services/compact/compact.ts`
- `src/utils/context.ts`
- `src/utils/tokens.ts`

## 快照修正与补充

- `docs/09-config-and-state.md` 更强调配置优先级，本章强调的是运行中的 token 预算调度。
- `other-ans/ch10.md` 的五层回收路径在当前快照里可以对应到 `compact` 相关目录和 `query.ts` 的预算控制逻辑。
- 当前快照仍保留 feature-gated 的压缩策略差异，因此不同构建目标下上下文回收路径不一定完全一致。

## 设计启示

- 对 Agent 来说，上下文预算管理更像内存管理，而不是简单的字符串截断。
- 分层压缩比一次性摘要更稳，因为它能把昂贵操作留到最后。
- 预算设计必须和系统提示、工具 schema、记忆系统一起考虑，而不是独立优化。
