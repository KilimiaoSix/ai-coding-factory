# 第 17 章：搜索工具——让 Agent 找到信息

## 问题定义

如果说文件编辑是 Agent 的手，那么搜索就是它的眼睛。没有高质量的搜索工具，模型只能靠猜测理解代码库，结果往往是误判结构、重复阅读或修改错误位置。

## 架构分析

当前快照把搜索拆成几类能力：内容搜索、文件名搜索、工具搜索，以及受环境影响的嵌入式/替代搜索路径。搜索工具看似简单，但实际承担了路径压缩、结果截断、排序和权限协作等职责，需要把底层搜索能力翻译成 LLM 容易使用的输入输出格式。

## 快照修正与补充

- `other-ans/ch17.md` 重点分析 ripgrep 封装方式，这与当前快照中的 Grep/Glob 设计高度一致。
- `docs/03-tool-system.md` 列出了 GrepTool 和 GlobTool，但没有强调搜索结果的“面向模型格式化”，本章将其补足。
- `src/tools.ts` 显示部分构建场景下可用嵌入式搜索替代通用 Grep/Glob，因此“搜索工具集合”并非绝对固定。

## 设计启示

- 搜索工具的价值不在于命令多强，而在于是否能用最少输入给模型返回最可消费的结果。
- 让搜索保持只读和高并发安全，是提升 Agent 工作效率的基础。
- 好的搜索输出会直接减少上下文浪费。
